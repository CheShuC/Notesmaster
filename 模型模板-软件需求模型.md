## 2. 用例详细描述

### UC-01: 创建文本笔记

| 项目 | 内容 |
|------|------|
| **用例 ID** | UC-01 |
| **用例名称** | 创建文本笔记 |
| **参与者** | 普通用户 |
| **前置条件** | 应用已启动，处于笔记列表界面 |
| **后置条件** | 新笔记保存到 SQLite 数据库，出现在列表中 |
| **主成功场景** | 1. 用户点击工具栏"新建笔记"按钮 → 2. 系统打开 `NoteEditActivity` → 3. 用户输入文字 → 4. 用户按返回键 → 5. `WorkingNote.saveNote()` 检查 `isWorthSaving()` → 6. 调用 `Note.syncNote()` → 7. 通过 `ContentResolver` 写入 `note` 和 `data` 表 → 8. 列表自动刷新显示新笔记 |
| **异常场景** | 若输入为空且笔记不在数据库中 → `isWorthSaving()` 返回 false → 静默返回不保存 |
| **优先级** | 高 |

### UC-02: 设置闹钟提醒

| 项目 | 内容 |
|------|------|
| **用例 ID** | UC-02 |
| **用例名称** | 设置闹钟提醒 |
| **参与者** | 普通用户 |
| **前置条件** | 已在笔记编辑界面，笔记非空 |
| **后置条件** | 闹钟注册到系统 AlarmManager，到期时锁屏弹窗 |
| **主成功场景** | 1. 用户点击"提醒我" → 2. `checkAlarmPermissions()` 检查 `POST_NOTIFICATIONS`（未授权→系统弹窗） → 3. 检查 `SCHEDULE_EXACT_ALARM`（未授权→引导到系统设置） → 4. 检查 `USE_FULL_SCREEN_INTENT`（未授权→引导到系统设置） → 5. 全部通过→弹出 `DateTimePickerCalendar` → 6. 用户选择日期和时间→确认 → 7. `AlarmManager.setExactAndAllowWhileIdle()` 注册闹钟 → 8. 更新 `NoteColumns.ALERT_DATE` |
| **异常场景** | 任意权限被拒绝→返回；笔记为空→Toast 提示"不能为空便签设置闹钟" |
| **优先级** | 高 |

### UC-03: 上传笔记到 GitHub

| 项目 | 内容 |
|------|------|
| **用例 ID** | UC-03 |
| **用例名称** | 上传笔记到 GitHub |
| **参与者** | 普通用户 |
| **前置条件** | 已配置 GitHub Token，网络正常 |
| **后置条件** | GitHub 仓库中存储所有笔记 JSON 文件（通话/短信除外） |
| **主成功场景** | 1. 用户点击"同步" → 2. AlertDialog 选择"上传" → 3. `GTaskSyncService` 启动前台服务 → 4. `clearAllRemote()` 删除远程所有文件 → 5. `resetLocalSyncMetadata()` 清除本地 gid → 6. 查询本地文件夹→ `createContainer()` → 7. 查询本地笔记→序列化 JSON→`createItem()` → 8. 更新本地 `gtask_id` → 9. 通知栏显示"新增 N 条" |
| **异常场景** | 网络错误→OkHttp 重试 3 次→仍失败显示"同步失败" |
| **优先级** | 高 |

### UC-04: 从 GitHub 下载笔记

| 项目 | 内容 |
|------|------|
| **用例 ID** | UC-04 |
| **用例名称** | 从 GitHub 下载笔记 |
| **参与者** | 普通用户 |
| **前置条件** | 已配置 GitHub Token，网络正常，远程仓库有笔记文件 |
| **后置条件** | 远程笔记同步到本地，本地独有笔记保留 |
| **主成功场景** | 1. 用户点击"同步" → 2. AlertDialog 选择"下载" → 3. `GTaskSyncService` 启动前台服务 → 4. `listContainers()` 列出远程目录 → 5. 逐个容器：`ensureLocalFolder()` → `listItems()` → 6. 逐个文件：`GET` 内容→`Task.setContentByRemoteJSON()` → `getLocalJSONFromContent()` → 7. `findLocalNoteByGid()`→有则更新，无则新建→`commit()` → 8. 通知栏显示"新增 N 条，更新 M 条" |
| **异常场景** | METADATA 容器自动跳过；JSON 中冲突 ID 自动剥离；远程无文件→"远程仓库无新文件可下载" |
| **优先级** | 高 |

```
    异常路径：
    任意步骤发生网络错误 → 重试 3 次 → 仍失败 → 通知栏"同步失败"
    任意步骤发生认证错误 → 通知栏"同步失败: 认证失败"
    用户取消 → 通知栏"同步已取消"
```
