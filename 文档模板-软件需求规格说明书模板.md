# 1. 引言

## 1.1 编写目的

本文档旨在完整定义 NotesMaster 便签应用的功能需求和非功能需求，为软件的后续设计、实现、测试和维护提供统一的参考基准。文档基于对 MiCode Notes（小米便签）开源代码的全面分析，明确了需要修复的原有缺陷和需要新增的功能特性。

## 1.2 读者对象

| 读者 | 用途 |
|------|------|
| 软件开发人员 | 理解功能需求，指导编码实现 |

主要功能和用途：文本笔记编辑与管理、通话记录自动便签、短信记录自动便签、文件夹分类管理、桌面小组件、闹钟提醒（含锁屏全屏通知）、全文搜索、SD 卡文本导出、GitHub 私有仓库云端同步（上传/下载）。

# 2. 软件的一般性描述

## 2.1 软件产品与其环境之间的关系

NotesMaster 作为 Android 移动应用，与以下外部环境交互：

```
┌──────────────────────────────────────────────────────────────┐
│                    Android 系统服务                           │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────────┐   │
│  │AlarmMgr  │ │NotifiMgr │ │Telephony │ │ContentResolver│   │
│  │(精确定时) │ │(通知弹出) │ │(通话状态) │ │(短信/通话记录) │   │
│  └──────────┘ └──────────┘ └──────────┘ └──────────────┘   │
└──────────────────────────────────────────────────────────────┘
         │              │            │             │
         ▼              ▼            ▼             ▼
    ┌──────────────────────────────────────────────────┐
    │              NotesMaster APK                     │
    │  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐  │
    │  │  UI  │ │Model │ │ Data │ │GTask │ │ Sync │  │
    │  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘  │
    │                      │                  │       │
    │                 SQLite DB           OkHttp     │
    └──────────────────────┼──────────────────┼──────┘
                           │                  │
                           ▼                  ▼
                    ┌──────────┐    ┌──────────────────┐
                    │ note.db  │    │  GitHub API      │
                    │ (本地)   │    │  api.github.com  │
                    └──────────┘    └────────┬─────────┘
                                             │
                                             ▼
                                    ┌──────────────────┐
                                    │  GitHub 私有仓库  │
                                    │  xiaomibianqian  │
                                    └──────────────────┘
```

| 外部环境 | 交互方式 | 依赖说明 |
|----------|---------|----------|
| Android AlarmManager | `setExactAndAllowWhileIdle()` | 精确定时闹钟，需 `SCHEDULE_EXACT_ALARM` 权限 |
| Android NotificationManager | `notify()` + `setFullScreenIntent()` | 通知和锁屏弹窗，需 `POST_NOTIFICATIONS` + `USE_FULL_SCREEN_INTENT` |
| Android Telephony | `PHONE_STATE` 广播 + `CallLog.Calls` ContentProvider | 通话便签，需 `READ_PHONE_STATE` + `READ_CALL_LOG` |
| Android ContentResolver | `content://sms/sent` + `SMS_RECEIVED` 广播 | 短信便签，需 `READ_SMS` + `RECEIVE_SMS` |
| Android Contacts | `ContactsContract` ContentProvider | 联系人姓名查询，需 `READ_CONTACTS` |
| SQLite (本地) | `NotesProvider` (authority: `micode_notes`) | 笔记和数据的本地持久化存储 |
| GitHub API | HTTPS REST (OkHttp 4.12) | 云端同步，需 GitHub PAT (`repo` scope) |
| GitHub 私有仓库 | Git 版本控制 | 笔记 JSON 文件存储和版本历史 |

## 2.2 限制与约束

| 类别 | 限制 | 影响 |
|------|------|------|
| 平台 | 仅支持 Android 7.0+ (API 24+) | 不支持 iOS 或 Web 端 |
| 同步 | 仅支持 GitHub 私有仓库 | 不支持 Google Tasks 或其他云服务 |
| 网络 | 同步需要互联网连接 | 离线时同步不可用 |
| 认证 | 需要用户自行创建 GitHub Token | 用户需有 GitHub 账号并了解 Token 创建流程 |
| API | GitHub API 认证用户 5000 次/小时 | 正常使用远超不到上限 |
| 存储 | 笔记以 JSON 文件存储在 GitHub | 文件名基于笔记首行（≤100 字符），多行笔记仅首行出现在文件名中 |
| 兼容 | 通话便签 API 34+ 限制 `PHONE_STATE` 广播 | 已通过 ContentObserver 备用方案解决 |
| 前台服务 | API 28+ 需 `FOREGROUND_SERVICE` 权限 | 已在 Manifest 声明 |
| 构建 | Apache HttpClient JAR 使用绝对路径 | 换机部署需调整 `build.gradle.kts` |

## 2.3 假设与前提条件

1. 用户设备运行 Android 7.0 或更高版本
2. 用户已授予应用所需的运行时权限（通话记录、短信、联系人、通知、精确定时闹钟、全屏通知）
3. 使用 GitHub 同步功能的用户：
   - 拥有 GitHub 账号
   - 已创建私有仓库 `xiaomibianqian`
   - 已生成具有 `repo` scope 的 Personal Access Token
4. 设备时间是准确的（闹钟功能依赖系统时间）
5. 应用有足够的存储空间存放本地 SQLite 数据库
6. 构建环境安装了 JDK 11+ 和 Android SDK API 36

### 用例详细描述

**UC-01：创建文本笔记**

| 项目 | 描述 |
|------|------|
| 参与者 | 普通用户 |
| 前置条件 | 应用已启动，处于笔记列表界面 |
| 主流程 | 1. 用户点击工具栏"新建笔记"按钮 → 2. 系统打开笔记编辑界面 → 3. 用户输入文字内容 → 4. 用户按返回键，系统自动保存笔记 → 5. 笔记出现在列表中 |
| 后置条件 | 笔记保存到 SQLite 数据库 |
| 异常流程 | 如果输入为空且笔记不存在于数据库，`isWorthSaving()` 返回 false，无声返回不保存 |

**UC-02：设置闹钟提醒**

| 项目 | 描述 |
|------|------|
| 参与者 | 普通用户 |
| 前置条件 | 笔记编辑界面已打开，笔记非空 |
| 主流程 | 1. 用户点击"提醒我" → 2. 系统检查 `POST_NOTIFICATIONS` 权限（未授权→系统弹窗） → 3. 系统检查 `SCHEDULE_EXACT_ALARM` 权限（未授权→引导到系统设置） → 4. 系统检查 `USE_FULL_SCREEN_INTENT` 权限（未授权→引导到系统设置） → 5. 全部通过→弹出日历时间选择器 → 6. 用户选择日期和时间→确认 → 7. 系统注册 AlarmManager 精确定时 |
| 后置条件 | 闹钟注册到系统 AlarmManager |
| 异常流程 | 任意权限被拒绝→返回，不设置闹钟 |

**UC-03：自动创建通话便签**

| 项目 | 描述 |
|------|------|
| 参与者 | 系统拨号器（外部触发） |
| 前置条件 | 已授予 `READ_CALL_LOG` 权限 |
| 主流程 | 1. 用户接听或拨出电话 → 2. 挂断后系统写入 `CallLog.Calls` 数据库 → 3. `CallLogObserver.onChange()` 触发 → 4. 查询最新通话记录 → 5. 时间戳去重 → 6. `createCallNote()` → `WorkingNote.createEmptyNote()` → `convertToCallNote()` → `setWorkingText()` → `saveNote()` |
| 后置条件 | 通话便签出现在"通话便签"文件夹中 |
| 异常流程 | 电话号码为空→设为 "private"；重复通话→去重跳过 |

**UC-04：上传笔记到 GitHub**

| 项目 | 描述 |
|------|------|
| 参与者 | 普通用户 |
| 前置条件 | 已配置 GitHub Token，网络正常 |
| 主流程 | 1. 用户点击工具栏"同步" → 2. 弹出对话框选择"上传（保存到远程）" → 3. `GTaskSyncService` 启动前台服务 → 4. `uploadAll()`：清空远程仓库 → 清除本地 gid → 创建 Default 容器 → 上传文件夹 → 逐个序列化笔记为 JSON → `PUT` 到 GitHub API → 更新本地 gid → 5. 通知栏显示"新增 N 条" |
| 后置条件 | GitHub 仓库中存储所有笔记的 JSON 文件，通话/短信文件夹被排除 |
| 异常流程 | 网络错误→自动重试 3 次→仍失败则显示"同步失败: ..." |

**UC-05：从 GitHub 下载笔记**

| 项目 | 描述 |
|------|------|
| 参与者 | 普通用户 |
| 前置条件 | 已配置 GitHub Token，网络正常，远程仓库有笔记文件 |
| 主流程 | 1. 用户点击工具栏"同步" → 2. 弹出对话框选择"下载（从远程读取）" → 3. `GTaskSyncService` 启动前台服务 → 4. `downloadAll()`：列出远程容器 → 逐个容器：确保本地文件夹存在 → 列出 `.json` 文件 → 逐个文件：`GET` 内容 → 解析 JSON → 根据 gid 查找本地笔记（存在则更新，不存在则新建）→ `commit()` 写入 SQLite → 5. 通知栏显示"新增 N 条，更新 M 条" |
| 后置条件 | 远程笔记完整同步到本地，本地独有笔记不被删除 |
| 异常流程 | `[MIUI_Notes]METADATA` 容器自动跳过；JSON 中有冲突 ID 自动剥离 |

# 4. 其它软件需求描述

## 4.1 性能要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 应用冷启动 | < 2 秒 | 从点击图标到笔记列表显示 |
| 笔记列表滚动 | 60 fps | RecyclerView/ListView 平滑滚动 |
| 笔记保存 | < 100ms | 从按返回键到 SQLite 写入完成 |
| 搜索建议响应 | < 500ms | 从输入字符到建议列表出现 |
| 同步（10 条笔记） | < 10 秒 | 含网络往返，正常网络环境 |
| 同步（100 条笔记） | < 60 秒 | 含网络往返，正常网络环境 |
| 内存占用 | < 100MB | 正常运行状态（不含图片） |
| APK 大小 | < 15MB | Debug 构建 |

## 4.2 设计约束

| 约束类别 | 具体要求 |
|----------|---------|
| **开发语言** | Java 11，不得使用 Kotlin 特性（保持与原始代码一致） |
| **构建系统** | Gradle 9.0.0 + AGP 9.0.0，使用 Kotlin DSL (`build.gradle.kts`) |
| **最低 SDK** | Android 7.0 (API 24) |
| **目标 SDK** | Android 15 (API 36) |
| **依赖库** | AndroidX appcompat 1.6.1、material 1.10.0、OkHttp 4.12；Apache HttpClient JAR 仅保留编译兼容 |
| **兼容性** | 需通过 `if (Build.VERSION.SDK_INT)` 版本分支处理 API 差异 |
| **安全性** | GitHub Token 存储在 SharedPreferences（仅本应用可读）；HTTPS 加密通信 |
| **架构约束** | 保持原始分层架构（UI → Model → Data + Tool + GTask + Sync + Widget），新增功能遵循现有包命名和代码风格 |
| **编码规范** | 成员变量 `m` 前缀，静态变量 `s` 前缀，常量 `UPPER_SNAKE_CASE`，TAG 使用 `class.getSimpleName()` |
