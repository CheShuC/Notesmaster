# "NotesMaster"软件系统的需求构思及描述

## 背景介绍

NotesMaster（原名 MiCode Notes）是一款源自小米 MiCode 开源社区（www.micode.net）的 Android 便签应用，采用 Apache License 2.0 开源协议发布。原始应用发布于 2010-2011 年，提供文本笔记编辑、通话记录便签、文件夹管理、桌面小组件、Google Tasks 云端同步等功能。

随着 Android 系统从 API 11 演进至 API 36，原始代码中大量依赖旧 API 的功能已完全失效——菜单按钮被系统折叠隐藏、闹钟因缺少 NotificationChannel 无法弹出通知、搜索因授权名不匹配无法返回结果、同步依赖被墙的 Google 服务。此外，原始代码缺少短信便签功能（仅预留了常量），通话便签依赖 MIUI 系统的私有 Intent 在标准 Android 上无法工作。

本项目（NotesMaster）是对该开源软件的全面维护和功能增强，目标是在现代 Android 系统（API 24-36）上恢复全部原有功能并新增短信便签和 GitHub 云端同步能力。

## 欲解决问题

本软件系统试图解决以下应用问题：

**1. 菜单和操作按钮不可见**

原始菜单文件缺少 `android:showAsAction` 属性，Android ActionBar 机制将未声明此属性的菜单项折叠入溢出菜单。用户看到的是空白工具栏，无法直接操作"新建笔记"、"搜索"、"删除"、"分享"、"设置提醒"等核心功能。

**2. 闹钟提醒完全不可用**

5 个独立的技术障碍导致闹钟从设置到弹出的全链路断裂：`AlarmAlertActivity` 使用 API 17 废弃的 `KeyguardLock`、`AlarmReceiver` 缺少 API 26+ 必需的 `NotificationChannel`、`DateTimePickerDialog` 仅支持时间选择无日期选择、`AlarmInitReceiver` 开机广播注册不正确、`POST_NOTIFICATIONS`/`SCHEDULE_EXACT_ALARM`/`USE_FULL_SCREEN_INTENT` 三个权限未在代码中运行时请求。

**3. 通话便签在标准 Android 上不工作**

原始代码依赖 MIUI 系统电话应用发送的 `ACTION_INSERT_OR_EDIT` Intent，标准 Android 拨号器不发送此 Intent。此外，API 34+ 限制了 `PHONE_STATE` 广播的接收者范围，传统 BroadcastReceiver 方案失效。

**4. 短信便签功能完全缺失**

原始代码预留了 `ID_SMS_RECORD_FOLDER = -4` 常量，但未实现短信便签的数据模型、广播接收器、UI 展示和编辑功能。

**5. 搜索功能 6 个问题导致搜索完全失效**

`searchSuggestAuthority` 授权名与实际 ContentProvider 不匹配、`ACTION_SEARCH` Intent 未被处理、搜索图标缺失密度变体、XML 格式错误、无搜索结果提示等。

**6. 同步功能依赖被墙的 Google 服务且架构过时**

原始同步基于 Google Tasks API，国内网络无法访问。同步按钮在菜单 XML 中被硬编码隐藏。`GTaskManager.java` 是一个 1000+ 行的复杂双向差量引擎（8 个哈希表、8 种同步动作、多阶段对账），维护极其困难。HTTP 通信使用已废弃的 Apache HttpClient。`GTaskSyncService` 未提升为前台服务，Android 8.0+ 系统在 5 秒后杀死进程导致同步闪退。

## 软件创意

基于 **Android 智能手机 + GitHub 私有仓库** 的组合，通过以下方案解决上述问题：

**菜单修复**：在所有菜单 XML 的 `<item>` 标签中添加 `android:showAsAction="ifRoom"` 属性，使操作按钮直接显示在 ActionBar 上。补充 `sub_folder.xml` 中缺失的返回和搜索按钮。

**闹钟系统重写**：采用现代 Android 通知架构——高优先级 Notification + `setFullScreenIntent()` + `NotificationChannel`。实现三层权限引导流程：标准运行时权限走系统弹窗（`POST_NOTIFICATIONS`），特殊权限引导到系统设置页面（`SCHEDULE_EXACT_ALARM`、`USE_FULL_SCREEN_INTENT`）。日期选择器从单一 `TimePicker` 升级为 `DatePicker`（日历网格）+ `TimePicker`（滚轮）组合。

**通话/短信便签双重保障**：针对 API 34+ 广播限制，采用 BroadcastReceiver（老设备）+ ContentObserver（所有设备）双重机制。`CallLogObserver` 监听 `CallLog.Calls` 数据库变化，`SmsReceiver` 监听 `SMS_RECEIVED` 广播，`SmsSentObserver` 监听 `content://sms/sent` 数据库变化，`onResume()` 前台补检确保不遗漏。去重逻辑复用 `DataUtils` 的电话号码+日期查询。

**搜索功能全链路修复**：修正 `searchSuggestAuthority`，实现 `handleSearchIntent()` 处理 `ACTION_SEARCH`，查询 `content://micode_notes/search` 返回 `AlertDialog` 结果列表。补充搜索图标密度变体和无结果提示。

**同步功能彻底重构**：

- 将同步后端从 Google Tasks API 替换为 **GitHub Contents API**，使用 GitHub 私有仓库 `xiaomibianqian` 存储笔记 JSON 文件
- 用 **OkHttp 4.12** 替换已废弃的 Apache HttpClient，添加自动重试拦截器（3 次指数退避）解决模拟器 BPF 网络栈丢包问题
- 将 1000+ 行的复杂双向差量引擎**完全重写**为 385 行的两个直线方法——`uploadAll()`（覆盖上传）和 `downloadAll()`（增量下载）
- 删除 `MetaData.java`（笔记 JSON 已自包含），删除全部 8 种同步动作常量和时间戳比较逻辑
- 新建 `sync/` 包实现**后端无关的同步客户端接口**——替换存储后端（如 WebDAV、S3）只需新增一个实现类
- `GTaskSyncService` 提升为前台服务，声明 `FOREGROUND_SERVICE_DATA_SYNC` 类型，防止系统杀死进程

### 要素关系

| 要素 | 类型 | 依赖 |
|------|------|------|
| Android 客户端 | 移动应用（APK） | Android 7.0+ (API 24+)，目标 API 36 |
| SQLite 数据库 | 本地存储 | Android 内置，无需额外安装 |
| GitHub API | 云端 REST 服务 | 互联网连接，GitHub 账号 |
| GitHub 私有仓库 | 云端文件存储 | 用户创建，存储笔记 JSON 文件 |
| OkHttp 4.12 | HTTP 客户端库 | Gradle 依赖管理，自动下载 |
| Apache HttpClient JAR | 编译兼容（仅保留引用） | 本地绝对路径 `D:\CODE\develop\` |

### 部署方式

1. **构建**：`./gradlew assembleDebug` 在 Android Studio 或命令行编译生成 APK
2. **安装**：通过 ADB 或应用商店安装到 Android 设备
3. **首次配置**：启动后授予通话记录/短信/联系人/通知等运行时权限
4. **同步配置**：
   - 在 GitHub 创建私有仓库 `xiaomibianqian`
   - 生成 Personal Access Token（`repo` scope）
   - 在应用设置中粘贴 Token → 测试连接 → 保存
5. **运行环境**：Android 7.0 (API 24) 至 Android 15 (API 36)，支持模拟器和真机

## 软件系统的功能描述

### 1. 笔记编辑和管理

- **文本笔记**：创建、编辑、删除文本笔记，支持清单模式（✓/☐ 标记）
- **字体大小**：4 档可调（小/中/大/超大）
- **背景颜色**：5 色可选（黄/蓝/白/绿/红），支持随机颜色
- **文件夹管理**：创建、重命名、删除文件夹，笔记在文件夹间移动
- **批量操作**：多选 → 批量删除/移动到文件夹
- **桌面小组件**：2×1 和 4×1 两种尺寸，点击直接进入笔记

### 2. 通话便签自动创建

- **来电便签**：`CallLogObserver` 监听通话记录数据库，检测到来电后自动创建便签
- **去电便签**：同上，拨出电话后自动创建便签
- **展示**：显示联系人姓名（需通讯录权限）、电话号码、通话时间
- **去重**：根据电话号码 + 通话日期查询，防止重复创建
- **双重保障**：BroadcastReceiver（API < 34）+ ContentObserver（全部 API）

### 3. 短信便签自动创建

- **收信便签**：`SmsReceiver` 监听 `SMS_RECEIVED` 广播，解析 PDU 创建便签
- **发信便签**：`SmsSentObserver` 监听 `content://sms/sent` 数据库，发送后自动创建
- **展示**：显示联系人姓名、短信正文预览
- **去重**：根据电话号码 + 短信日期查询

### 4. 闹钟提醒

- **日历时间选择器**：`DatePicker`（日历网格）+ `TimePicker`（滚轮），支持选择任意日期和时间
- **权限引导**：自动检测并引导授权 `POST_NOTIFICATIONS`、`SCHEDULE_EXACT_ALARM`、`USE_FULL_SCREEN_INTENT`
- **精确定时**：`AlarmManager.setExactAndAllowWhileIdle()`，Doze 模式不延迟
- **锁屏弹窗**：高优先级 Notification + `setFullScreenIntent()`，锁屏状态下全屏显示
- **开机恢复**：`AlarmInitReceiver` 监听 `BOOT_COMPLETED`，重启后自动恢复未过期闹钟

### 5. 全文搜索

- **搜索建议**：随输入实时显示匹配的笔记标题（通过 `searchSuggestAuthority` 查询 ContentProvider）
- **搜索结果**：回车提交后弹出 AlertDialog 结果列表，点按打开对应笔记
- **搜索高亮**：打开笔记时自动滚动到搜索词位置并高亮
- **覆盖范围**：搜遍全部笔记正文（文本/通话/短信），通过 SQLite 触发器保持 snippet 同步
- **友好提示**：无结果时显示"没有找到便签"

### 6. GitHub 云端同步

- **上传（覆盖模式）**：清空远程仓库 → 全量上传所有笔记（通话/短信便签自动排除）→ 笔记以 JSON 文件形式存储在 GitHub 仓库中
- **下载（增量模式）**：列出远程目录 → 逐个下载 JSON 文件 → 已有笔记更新，新笔记新建，本地独有笔记保留不删
- **Token 管理**：GitHub Personal Access Token 配置引导页（中文分步教程），支持连接测试和管理
- **前台服务**：同步期间状态栏显示持久通知，防止系统杀进程
- **网络容错**：OkHttp 重试拦截器（3 次指数退避：1s/2s/4s），应对模拟器 BPF 丢包等问题
- **数据安全**：所有笔记存储在用户自己的 GitHub 私有仓库中，仅 Token 持有者可访问

### 7. 其他功能

- **SD 卡导出**：将全部笔记导出为文本文件到 `/MIUI/notes/` 目录
- **分享**：通过系统分享菜单将笔记内容发送到其他应用
- **发送到桌面**：为单条笔记创建桌面快捷方式
- **三语支持**：英文 / 简体中文 / 繁体中文

## 可行性及潜在风险

### 技术可行性

| 技术点 | 可行性 | 说明 |
|--------|--------|------|
| Android API 24-36 兼容 | ✅ 可行 | 使用 AndroidX 兼容库，`if (Build.VERSION.SDK_INT)` 分支处理版本差异 |
| GitHub Contents API | ✅ 可行 | RESTful 接口，认证用户 5000 次/小时，笔记同步绰绰有余 |
| OkHttp 4.12 | ✅ 可行 | 成熟稳定的 HTTP 库，支持 Android 5.0+ |
| 前台服务 | ✅ 可行 | 已声明所需权限并处理 API 28/34 版本差异 |
| SQLite | ✅ 可行 | Android 内置，无需额外依赖 |
| 权限请求 | ✅ 可行 | 使用 AndroidX `ActivityResultContracts` 和标准权限框架 |

### 条件约束

- GitHub 同步需要互联网连接，离线时同步按钮可点击但会因网络错误失败（已通过重试机制缓解间歇性故障）
- 用户需自行创建 GitHub 仓库并生成 Token——应用中提供了分步中文引导
- Apache HttpClient JAR 文件使用绝对路径引用（`D:\CODE\develop\`），换机部署需调整 `build.gradle.kts`
- 模拟器存在已知 BPF 网络栈 bug（`E2BIG` 错误），已通过 OkHttp 重试拦截器缓解——真机上无此问题

### 潜在风险和应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| GitHub API 限速（5000次/小时） | 大量笔记初次同步可能触发 | 正常使用远超不到上限（每次同步约 20-50 次请求）；后续可加条件请求 `If-None-Match` |
| Token 泄露 | 仓库数据被非授权访问 | Token 存储在 SharedPreferences，仅本应用可读；用户可随时在 GitHub 吊销 Token |
| 同步冲突（两设备同时修改） | GitHub API 返回 409 Conflict | `GitHubSyncClient` 将 409 转换为 `SyncException.CONFLICT`，同步失败并提示用户 |
| 旧 METADATA 容器残留 | 从旧版本升级后远程仓库有冗余目录 | `downloadAll()` 自动跳过 `[MIUI_Notes]METADATA` 容器；`uploadAll()` 的 `clearAllRemote()` 会清除 |
| 文件名冲突（两笔记首行相同） | 后上传的覆盖先上传的 | 当前概率极低（首行 + 100 字符截断）；可后续通过在文件名追加数字解决 |
| Gradle/AGP 版本兼容 | 项目使用 AGP 9.0.0 + Gradle 9.2.1 | 新版 IDE 可能需要调整；已在 `CLAUDE.md` 和部署文档中记录兼容配置 |
