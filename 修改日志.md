### 4. 新增 `checkAlarmPermissions()` — 三层检查

```java
private boolean checkAlarmPermissions() {
    // 检查 1: POST_NOTIFICATIONS (API 33+)
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU) {
        if (checkSelfPermission(Manifest.permission.POST_NOTIFICATIONS)
                != PackageManager.PERMISSION_GRANTED) {
            requestPermissions(new String[]{Manifest.permission.POST_NOTIFICATIONS},
                    REQUEST_CODE_POST_NOTIFICATIONS);
            return false;
        }
    }

    // 检查 2: SCHEDULE_EXACT_ALARM (API 31+)
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.S) {
        AlarmManager am = (AlarmManager) getSystemService(Context.ALARM_SERVICE);
        if (!am.canScheduleExactAlarms()) {
            showPermissionGuideDialog(..., Settings.ACTION_REQUEST_SCHEDULE_EXACT_ALARM);
            return false;
        }
    }

    // 检查 3: USE_FULL_SCREEN_INTENT (API 34+)
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.UPSIDE_DOWN_CAKE) {
        NotificationManager nm = ...;
        if (!nm.canUseFullScreenIntent()) {
            showPermissionGuideDialog(..., Settings.ACTION_MANAGE_APP_USE_FULL_SCREEN_INTENT);
            return false;
        }
    }

    return true;
}
```

## 修改 15：添加 CallReceiver 监听系统通话，自动创建通话便签

**日期**：2026-05-31

**问题描述**：

通话便签依赖于 MIUI 系统电话应用在挂断后发送 `ACTION_INSERT_OR_EDIT` Intent，但标准 Android 拨号器（包括模拟器）不发送此 Intent。导致模拟来电后无法自动创建通话便签，"通话便签"文件夹始终为空。

**方案**：新增 `CallReceiver`（BroadcastReceiver），监听系统 `PHONE_STATE` 和 `NEW_OUTGOING_CALL` 广播，通话结束后自动创建通话便签。

**修改文件**：

| 文件 | 路径 | 操作 |
|------|------|------|
| CallReceiver.java | `Notesmaster/app/src/main/java/net/micode/notes/ui/CallReceiver.java` | **新建** |
| AndroidManifest.xml | `Notesmaster/app/src/main/AndroidManifest.xml` | 权限 + Receiver 注册 |
| NotesListActivity.java | `Notesmaster/app/src/main/java/net/micode/notes/ui/NotesListActivity.java` | 运行时权限请求 |

**修改内容**：

### 1. 新建 `CallReceiver.java`

```java
public class CallReceiver extends BroadcastReceiver {
    private static String sPendingNumber;

    @Override
    public void onReceive(Context context, Intent intent) {
        String action = intent.getAction();
        // 来电：响铃时记录号码，挂断后创建便签
        if (PHONE_STATE_CHANGED.equals(action)) {
            if (RINGING.equals(state)) {
                sPendingNumber = intent.getStringExtra(EXTRA_INCOMING_NUMBER);
            } else if (IDLE.equals(state) && sPendingNumber != null) {
                createCallNote(context, sPendingNumber);
                sPendingNumber = null;
            }
        }
        // 去电：拨号时立即创建便签
        else if (NEW_OUTGOING_CALL.equals(action)) {
            String number = intent.getStringExtra(Intent.EXTRA_PHONE_NUMBER);
            createCallNote(context, number);
        }
    }

    private void createCallNote(Context context, String phoneNumber) {
        // 去重检查
        long noteId = DataUtils.getNoteIdByPhoneNumberAndCallDate(...);
        if (noteId > 0) return;

        // 使用 WorkingNote 标准流程创建
        WorkingNote note = WorkingNote.createEmptyNote(context,
                Notes.ID_CALL_RECORD_FOLDER,
                AppWidgetManager.INVALID_APPWIDGET_ID,
                Notes.TYPE_WIDGET_INVALIDE,
                ResourceParser.getDefaultBgId(context));
        note.convertToCallNote(phoneNumber, System.currentTimeMillis());
        note.saveNote();
    }
}
```

### 2. `AndroidManifest.xml` — 新增权限 + 注册 Receiver

**新增权限**：
```diff
+<uses-permission android:name="android.permission.READ_PHONE_STATE" />
+<uses-permission android:name="android.permission.PROCESS_OUTGOING_CALLS" />
```

| 权限 | 用途 |
|------|------|
| `READ_PHONE_STATE` | 读取来电号码（`EXTRA_INCOMING_NUMBER`） |
| `PROCESS_OUTGOING_CALLS` | 接收去电广播（`NEW_OUTGOING_CALL`） |

**注册 Receiver**（放在 `<application>` 最顶部，确保尽早初始化）：
```xml
<receiver
    android:name=".ui.CallReceiver"
    android:exported="true">
    <intent-filter>
        <action android:name="android.intent.action.PHONE_STATE" />
        <action android:name="android.intent.action.NEW_OUTGOING_CALL" />
    </intent-filter>
</receiver>
```

### 3. `NotesListActivity.java` — 运行时权限请求

`READ_PHONE_STATE` 是运行时权限（API 23+），App 首次启动时弹窗请求：

```diff
+import android.Manifest;
+import android.content.pm.PackageManager;
+import android.os.Build;
+

      ...
      setAppInfoFromRawRes();

  }

+private void requestPhonePermission() {
+    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.M) {
+        if (checkSelfPermission(Manifest.permission.READ_PHONE_STATE)
+                != PackageManager.PERMISSION_GRANTED) {
+            requestPermissions(new String[]{
+                    Manifest.permission.READ_PHONE_STATE

+}
```

**工作流程**：

```
来电:
  系统拨号器响铃 → PHONE_STATE=RINGING → CallReceiver 记录号码
  用户挂断      → PHONE_STATE=IDLE    → CallReceiver 创建便签

去电:
  用户拨号      → NEW_OUTGOING_CALL   → CallReceiver 立即创建便签
```

**与现有代码的兼容性**：

- 使用 `WorkingNote.createEmptyNote()` → `convertToCallNote()` → `saveNote()` 标准流程
- `convertToCallNote()` 设置 `PARENT_ID = ID_CALL_RECORD_FOLDER`，便签自动归入通话便签文件夹
- 去重逻辑复用 `DataUtils.getNoteIdByPhoneNumberAndCallDate()`
- 联系人姓名查询复用 `Contact.getContact()`
- 列表显示复用 `NoteItemData` / `NotesListItem` 的通话便签展示逻辑

**效果**：

- 模拟器（及任何标准 Android 拨号器）来电/去电后自动创建通话便签
- 通话便签文件夹自动出现在主列表（有便签时才显示）
- 便签中包含电话号码和通话时间，自动匹配联系人姓名
- 与 MIUI 原始通话便签功能一致，仅入口不同

> **注**：修改 15 在 API 36 上不生效，原因如下。修改 16 添加了 CallLog ContentObserver 后备方案解决。

## 修改 16：CallReceiver 增强日志 + CallLog ContentObserver 后备方案

**日期**：2026-05-31

**问题描述**：

修改 15 的 `CallReceiver`（监听 `PHONE_STATE` 广播）在模拟器拨号后没有触发，通话便签文件夹始终为空。

**根因分析**：

`PHONE_STATE` 广播在 API 34+（Android 14+）受到严格限制：
- 系统不再向所有注册了 `PHONE_STATE` intent-filter 的应用发送广播
- 仅默认电话应用和系统组件收到该广播
- 即使应用有 `READ_PHONE_STATE` 权限也不行

**方案**：保留 `CallReceiver`（在老设备上仍可能生效），新增 `CallLogObserver`——监听系统通话记录数据库 `CallLog.Calls` 的变化。拨号后通话记录必然写入数据库，这是系统级保证，不受广播限制影响。

**修改文件**：

| 文件 | 路径 |
|------|------|
| CallReceiver.java | `Notesmaster/app/src/main/java/net/micode/notes/ui/CallReceiver.java` |
| AndroidManifest.xml | `Notesmaster/app/src/main/AndroidManifest.xml` |
| NotesListActivity.java | `Notesmaster/app/src/main/java/net/micode/notes/ui/NotesListActivity.java` |

**修改内容**：

### 1. `CallReceiver.java` — 添加调试日志

- 所有关键分支加 `Log.w(TAG, ...)`（warning 级别，不用 debug 也可见）
- `createCallNote()` 改为 `static` 方法，可供 `NotesListActivity` 直接调用
- 每步都有日志输出，便于在 Logcat 中追踪

### 2. `AndroidManifest.xml` — 新增 READ_CALL_LOG 权限

```diff
+<uses-permission android:name="android.permission.READ_CALL_LOG" />
```

`READ_CALL_LOG` 允许读取系统通话记录数据库，是 `CallLogObserver` 的必需权限。

### 3. `NotesListActivity.java` — 核心变更

**新增字段**：
```java
private long mLastProcessedCallId;
private CallLogObserver mCallLogObserver;
```

**新增 `onCreate` 中注册**：
```java

```

**新增 `onResume` 中补检**：回到前台时扫描一遍通话记录，处理后台新增的通话。

**新增 `onDestroy` 中注销**。

**新增 `CallLogObserver` 内部类**：
```java
private class CallLogObserver extends ContentObserver {
    @Override
    public void onChange(boolean selfChange) {
        checkCallLogForNewCalls();
    }
}
```

**新增 `checkCallLogForNewCalls()`**：
```java
private void checkCallLogForNewCalls() {
    // 权限检查
    // 查询 CallLog.Calls 中最新一条记录
    // 如果 callId > mLastProcessedCallId（新通话）→ 调用 CallReceiver.createCallNote()
    // 更新 mLastProcessedCallId 避免重复处理
}
```

**更新 `requestPhonePermission()`**：同时请求 `READ_PHONE_STATE` 和 `READ_CALL_LOG` 两颗权限。

**工作原理**：

```
用户拨号/接听 → 挂断
  → 系统写入 CallLog.Calls 数据库
    → CallLogObserver.onChange() 被触发
      → checkCallLogForNewCalls()
        → 查询最新 call log entry
        → ID 比对去重（mLastProcessedCallId）
        → CallReceiver.createCallNote(context, phoneNumber)
          → WorkingNote.createEmptyNote → convertToCallNote → saveNote ✅

回到主列表 → onResume()
  → checkCallLogForNewCalls() 再次扫描
    → 补检后台新增的通话 ✅
```

**双重保障**：

| 机制 | API 限制 | 触发方式 |
|------|----------|---------|
| `CallReceiver` (PHONE_STATE) | API 34+ 不发给普通应用 | 系统广播 |
| `CallLogObserver` (ContentObserver) | 无限制，所有 API 可用 | 数据库变更通知 |

**效果**：

- 模拟器拨号后挂断，`CallLogObserver` 检测到通话记录新增 → 自动创建通话便签
- App 从后台回到前台时自动扫描是否有新增通话记录
- 去重机制防止重复创建
- `CallReceiver` 保留作为老设备上的额外保障

## 修改 19：发送短信（去电短信）也会创建短信便签

**日期**：2026-05-31

**问题描述**：

修改 18 实现的 `SmsReceiver` 只监听了 `SMS_RECEIVED`（收短信），但用户发送（去电）的短信不会触发此广播。需要同时监听发送的短信，确保发件箱中的短信也会自动创建便签。

**方案**：

Android 没有 `SMS_SENT` 之类的标准系统广播可用于第三方应用检测发送短信。最可靠的方式是通过 `ContentObserver` 监听系统短信数据库 `content://sms/sent` 的变化，并在 `onResume()` 时进行补检。

**修改文件**：

| 文件 | 路径 | 操作 |
|------|------|------|
| NotesListActivity.java | `Notesmaster/app/src/main/java/net/micode/notes/ui/NotesListActivity.java` | 修改 |

**修改内容**：

### 1. 新增 import

```diff
+import android.database.ContentObserver;
+import android.net.Uri;
```

### 2. 新增字段 `mSmsSentObserver`

```java
private SmsSentObserver mSmsSentObserver;
```

### 3. `onCreate()` — 注册 ContentObserver

```java
mSmsSentObserver = new SmsSentObserver();
getContentResolver().registerContentObserver(
        Uri.parse("content://sms"), true, mSmsSentObserver);
```

监听 `content://sms` 整个短信数据库（`notifyForDescendants=true`），确保发送短信后立即触发。

### 4. 新增 `onDestroy()` — 注销 ContentObserver

```java

    super.onDestroy();
    if (mSmsSentObserver != null) {
        getContentResolver().unregisterContentObserver(mSmsSentObserver);
    }
}
```

### 5. `onResume()` — 前台补检

```diff

     super.onResume();
     checkCallLogForNewCalls();
+    checkSentSmsForNewMessages();
     startAsyncNotesListQuery();
 }
```

### 6. 新增 `checkSentSmsForNewMessages()` 方法

```java
private void checkSentSmsForNewMessages() {
    // 权限检查: READ_SMS
    // 查询 content://sms/sent，按 date ASC 排序
    // 用 last_sms_check (SharedPreferences) 去重
    // 遍历结果 → SmsReceiver.createSmsNote()
    // 更新 last_sms_check 时间戳
}
```

与 `checkCallLogForNewCalls()` 使用完全相同的模式：
- `SharedPreferences` 存储上次检查时间戳
- 查询 `content://sms/sent` 中 `date > lastCheckedTime` 的记录
- 调用 `SmsReceiver.createSmsNote()` 创建短信便签（自带去重）
- Toast 提示创建数量

+    Manifest.permission.READ_SMS
 }, REQUEST_CODE_READ_PHONE_STATE);

### 8. 新增 `SmsSentObserver` 内部类

```java
private class SmsSentObserver extends ContentObserver {
    public SmsSentObserver() {
        super(null);
    }

    @Override
    public void onChange(boolean selfChange) {
        Log.w(TAG, "SmsSentObserver: onChange triggered");
        checkSentSmsForNewMessages();
    }
}
```

### 双重保障机制

| 机制 | 触发方式 | 负责 |
|------|----------|------|
| `SmsReceiver` | `SMS_RECEIVED` 广播 | 收短信 |
| `SmsSentObserver` + `onResume()` | ContentObserver / 前台补检 | 发短信 |

**工作流程**：

```
用户发送短信
  → 系统写入 content://sms/sent 数据库
    → SmsSentObserver.onChange() 触发
      → checkSentSmsForNewMessages()
        → 查询 date > last_sms_check 的新记录
        → SmsReceiver.createSmsNote() 创建便签 ✅

App 从后台回到前台 → onResume()
  → checkSentSmsForNewMessages() 补检
    → 处理后台期间发送的所有短信 ✅
```

**效果**：

- 发送短信后，`ContentObserver` 实时检测并创建短信便签
- App 切回前台时自动补检后台期间的发送记录
- 时间戳去重，不会重复创建
- 与收短信便签使用相同的 `SmsReceiver.createSmsNote()` 方法，数据格式完全一致
