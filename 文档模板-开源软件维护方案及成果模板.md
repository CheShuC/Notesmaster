# **开源软件的维护方案及成果**

撰写人：车abcd

## 开源软件简介

NotesMaster（原名 MiCode Notes）是一款源自小米 MiCode 开源社区（www.micode.net）的 Android 便签应用，采用 Apache License 2.0 开源协议发布。该应用提供文本笔记编辑、通话记录便签、短信记录便签、文件夹分类管理、桌面小组件（Widget）、闹钟提醒、全文搜索、SD 卡备份导出以及通过 GitHub 私有仓库进行云端同步等功能。

| 属性 | 值 |
|---|---|
| **项目名称** | Notes-master (NotesMaster) |
| **开发者** | MiCode Open Source Community |
| **出处** | https://github.com/MiCode/Notes |
| **编程语言** | Java 11 |
| **构建系统** | Gradle 9.0.0 (Kotlin DSL) |
| **最低 SDK** | 24 (Android 7.0) |
| **目标 SDK** | 36 (Android 15) |
| **代码规模** | 约 11,700 行 Java 代码，45 个源文件，112 个资源文件 |
| **包结构** | 10 个主要包：data、model、tool、ui、widget、gtask.data、gtask.exception、gtask.remote、sync、sync.github |

## 软件维护需求描述

### 2.1 开源软件分析

通过对原始 MiCode Notes 开源代码的全面分析（含代码审查报告），发现该软件存在以下不足和待完善内容：

**（1）界面菜单显示缺陷**

原始代码中多处菜单配置文件缺少 `android:showAsAction` 属性。在 Android 3.0+（API 11+）的 ActionBar 机制中，缺少此属性的菜单项默认进入溢出菜单（overflow menu），用户需点击"更多"按钮才能看到操作选项。这导致：

- 主列表界面（`note_list.xml`）的 5 个菜单项全部隐藏：新建文件夹、导出文本、同步、设置、搜索
- 笔记编辑界面（`note_edit.xml`）的 8 个按钮全部隐藏：新建、删除、字体大小、清单模式、分享、发送到桌面、提醒、删除提醒
- 子文件夹界面（`sub_folder.xml`）缺少返回按钮（`menu_back`）和搜索按钮（`menu_search`），且唯一的"新建笔记"项也因缺少 `showAsAction` 而被隐藏

**（2）短信便签功能缺失**

原始代码中已预留 `ID_SMS_RECORD_FOLDER = -4` 常量，但未实现任何短信便签功能：没有 SmsReceiver 广播接收器、没有短信数据模型（SmsNote）、没有短信便签的 UI 展示和编辑功能、在 `ListEditState` 枚举和 `ROOT_FOLDER_SELECTION` 中均未包含短信文件夹。通话记录便签虽已实现但因依赖 MIUI 系统电话应用发送的 `ACTION_INSERT_OR_EDIT` Intent，在标准 Android 设备上无法工作。

**（3）闹钟提醒功能不可用（5 个崩溃点）**

- `AlarmAlertActivity` 使用了 Android API 17 已废弃的 `KeyguardLock` 和新版权限模型下失效的 `WakeLock` 模式，无法在锁屏状态下正常显示
- `AlarmReceiver` 未创建 `NotificationChannel`（API 26+ 必需），导致通知无法发出
- `DateTimePickerDialog` 仅支持时间选择，缺少日期选择功能
- `AlarmInitReceiver` 位置错误（放在 widget 包中）且未正确注册开机广播
- 三个关键权限未在代码中运行时请求：`POST_NOTIFICATIONS`（API 33+）、`SCHEDULE_EXACT_ALARM`（API 31+）、`USE_FULL_SCREEN_INTENT`（API 34+），导致闹钟界面无法弹出

**（4）搜索功能不可用（6 个问题）**

- `searchable.xml` 中 `searchSuggestAuthority="notes"` 与实际 ContentProvider 授权名 `"micode_notes"` 不匹配，搜索建议功能完全失效
- `ACTION_SEARCH` Intent 未在 `NoteEditActivity` 中被处理，用户回车提交搜索后 Activity 直接关闭
- `sub_folder.xml` 缺少搜索菜单项，进入子文件夹后无法触发搜索
- `AndroidManifest.xml` 存在 XML 格式错误（`</intent-filter >>` 多余字符）
- 缺少"无搜索结果"的用户提示字符串资源
- `search_result.png` 图标仅存在于 `drawable-hdpi` 一个密度目录

**（5）代码质量问题**

代码审查发现的 10 项主要质量问题：
- `NotesProvider.getType()` 方法未实现，返回 null
- `increaseNoteVersion` 方法存在 SQL 参数非参数化拼接风险
- `Contact.getContact()` 中 Cursor 泄露风险
- 多处常量拼写错误（如 `ID_TEMPARAY_FOLDER`、`ID_TRASH_FOLER`）
- 线程安全隐患（HashMap 并发访问）
- 广泛捕获 Exception 吞噬异常信息
- 缺少输入参数 null 校验
- JSON 类型不匹配时缺乏容错

**（6）同步功能依赖被墙的 Google 服务且实现不完整**

- 原始同步基于 Google Tasks API（`gtask/` 包），依赖 `@gmail.com` 账号登录，国内网络环境无法访问
- `note_list.xml` 中同步按钮被硬编码 `android:visible="false"`，用户界面完全看不到同步入口
- `GTaskClient.java` 使用已废弃的 Apache HttpClient 4.5.14 进行 HTTP 通信，不支持现代 Android API
- 代码中有 TODO 注释 "后续迁移至阿里云同步服务"，表明开发者已知此问题未解决
- 原双向同步引擎（`GTaskManager.java`，1000+ 行）结构复杂，维护 8 个内存哈希表，包含 8 种同步动作类型的分发逻辑，适配新的同步后端极其困难

### 2.2 开源软件维护需求

根据上述分析，确定以下 6 项维护需求：

**1. 修复隐藏的菜单栏**

在 `sub_folder.xml` 中补充返回按钮和搜索按钮，为所有菜单项添加 `android:showAsAction="ifRoom"` 属性，使工具栏按钮可见。

**2. 修复隐藏的便签操作按钮**

在 `note_edit.xml`、`call_note_edit.xml`、`note_list.xml`、`call_record_folder.xml` 等菜单文件中为所有菜单项添加 `showAsAction="ifRoom"` 属性，将隐藏的编辑操作按钮（删除、分享、字体、提醒等）显示在工具栏。

**3. 修复被隐藏的文件夹显示内容**

在 `NotesListActivity` 中新增 `SMS_RECORD_FOLDER` 状态支持，更新 `ROOT_FOLDER_SELECTION` 包含短信文件夹，使用户在列表中有短信便签时能看见并进入"短信便签"文件夹。

**4. 修复无法使用的闹钟提醒功能**

全面重写闹钟系统：将 `AlarmReceiver` 从直接启动 Activity 改为高优先级 Notification + FullScreenIntent 模式；为 `AlarmAlertActivity` 添加锁屏唤醒和全屏通知能力；在 `NoteEditActivity` 中实现三层权限检查流程；修复 `AlarmInitReceiver` 的开机重注册逻辑。

**5. 修复电话记录便签和短信记录便签**

新增 `CallReceiver`（BroadcastReceiver）+ `CallLogObserver`（ContentObserver）双重保障机制，使通话便签在标准 Android 设备上也能自动创建。仿照通话记录模型，完整实现短信便签的数据模型（`SmsNote`）、广播接收器（`SmsReceiver`）、ContentObserver（`SmsSentObserver`）、UI 展示和编辑功能。

**6. 修复无法使用的搜索栏**

将 `searchSuggestAuthority` 修正为 `micode_notes`；在 `NoteEditActivity` 中实现 `handleSearchIntent()` 方法处理 `ACTION_SEARCH`，查询数据库返回结果列表；为 `sub_folder.xml` 添加搜索菜单项；修复 `AndroidManifest.xml` XML 格式错误；添加"无搜索结果"提示；复制搜索图标到全部密度目录。

**7. 重构同步功能 — Google Tasks → GitHub 私有仓库**

将同步后端从 Google Tasks API 替换为 GitHub Contents API，通过 GitHub 私有仓库 `xiaomibianqian` 存储笔记 JSON 文件。同步机制从复杂双向差量引擎完全重写为两个直接方法——`uploadAll()`（覆盖上传）和 `downloadAll()`（增量下载），始终排除通话/短信文件夹。删除 `MetaData.java`，移除 8 种同步动作常量和时间戳比较逻辑。使用 OkHttp 4.12 替换已废弃的 Apache HttpClient，添加自动重试拦截器（3 次指数退避）。新建 `sync/` 包实现后端无关的同步客户端接口。

#### 主要体系结构调整说明

**1. Data 层调整**

- `Notes.java`：新增 `SmsNote` 内部数据模型类，定义短信便签的数据列映射（SMS_DATE → DATA1, PHONE_NUMBER → DATA3, SMS_BODY → DATA4）和 MIME 类型常量
- `NotesDatabaseHelper.java`：数据库版本从 4 升级至 5，新增 `upgradeToV5()` 方法创建短信系统文件夹（`ID_SMS_RECORD_FOLDER = -4`）
- `DataUtils.java`：新增 2 个短信查询方法用于去重和展示

**2. Model 层调整**

- `Note.java`：内部类 `NoteData` 新增短信数据存储（`mSmsDataId`、`mSmsDataValues`）和 `setSmsData()` 方法，`pushIntoContentResolver()` 中新增短信类型的数据持久化逻辑
- `WorkingNote.java`：新增 `convertToSmsNote()` 方法，仿照 `convertToCallNote()` 实现，将工作笔记转换为短信便签类型

**3. UI 层调整**

- `NotesListActivity.java`：
  - `ListEditState` 枚举新增 `SMS_RECORD_FOLDER` 状态
  - `ROOT_FOLDER_SELECTION` 新增 `ID_SMS_RECORD_FOLDER` 的 OR 条件
  - 新增 `CallLogObserver` 内部类（监听通话记录数据库变化）
  - 新增 `SmsSentObserver` 内部类（监听发送短信数据库变化）
  - 新增 `checkCallLogForNewCalls()` 和 `checkSentSmsForNewMessages()` 补检方法
  - 新增 `registerCallLogObserver()` 和权限请求逻辑
- `NoteEditActivity.java`：
  - 新增 `checkAlarmPermissions()` 三层权限检查方法
  - 新增 `showPermissionGuideDialog()` 引导用户开启系统权限
  - 新增 `onRequestPermissionsResult()` 处理运行时权限回调
  - 新增 `handleSearchIntent()` 处理 `ACTION_SEARCH` Intent
  - `setReminder()` 前增加权限前置检查
- `CallReceiver.java`：新增的 BroadcastReceiver，监听 `PHONE_STATE` 和 `NEW_OUTGOING_CALL` 广播
- `SmsReceiver.java`：新增的 BroadcastReceiver，监听 `SMS_RECEIVED` 广播并解析 PDU
- `AlarmReceiver.java`：从直接 startActivity 重写为 Notification + FullScreenIntent 模式
- `AlarmAlertActivity.java`：添加 `DataExtractionRules` 兼容和 Notification 展示逻辑

## 维护后的开源软件

### 维护后的开源软件功能及演示

维护后的 NotesMaster 应用具备以下完整功能：

**（1）完整的笔记编辑工具栏**

进入笔记编辑界面后，工具栏直接显示所有操作按钮：新建（📝）、删除（🗑）、字体大小（A）、清单模式（☑）、分享（↗）、发送到桌面（📌）、提醒（⏰）。用户可以一键操作，无需进入溢出菜单。

**（2）完整的文件夹导航工具栏**

在根列表和子文件夹中，工具栏直接显示：新建文件夹（📁）、导出文本（💾）、设置（⚙）、搜索（🔍）。进入子文件夹后工具栏显示：返回（←）、新建笔记（📝）、搜索（🔍）。

**（3）闹钟提醒功能**

用户点击"提醒我"后，系统自动检查并引导用户授权所需权限：
- `POST_NOTIFICATIONS`：系统标准权限弹窗
- `SCHEDULE_EXACT_ALARM`：引导到系统设置页面开启"闹钟和提醒"权限
- `USE_FULL_SCREEN_INTENT`：引导到系统设置页面开启"全屏通知"权限

全部授权后，用户设置闹钟时间，到时间后手机锁屏状态下也会弹出全屏通知，显示便签内容，用户可点击"去看看"直接进入笔记。

**（4）通话便签自动创建**

用户在标准 Android 手机上拨打或接听电话后，返回应用即可看到"通话便签"文件夹出现在列表中。文件夹内有本次通话的便签，显示联系人姓名（需通讯录权限）和通话时间。点击文件夹查看通话列表，每条便签正文显示通话时间。

**（5）短信便签自动创建**

用户收到短信后，`SmsReceiver` 自动创建短信便签，包含发件人姓名（或号码）、短信时间和短信正文预览。用户发送短信后，`SmsSentObserver` 同样自动创建出发短信便签。返回应用后可看到"短信便签"文件夹和其中所有短信记录。

**（6）全文搜索功能**

用户点击搜索图标后，输入搜索关键词，系统实时显示匹配的笔记标题作为搜索建议。点按建议直接打开笔记并高亮搜索词。按回车提交后显示结果列表对话框，点按任意结果打开对应笔记。无匹配结果时友好提示"没有找到便签"。

**（7）GitHub 云端同步**

用户通过 GitHub Personal Access Token 配置同步后，可从主列表菜单或设置页触发同步：

- **上传**：点击同步 → 选择"上传（保存到远程）"→ 清空远程仓库 → 全量上传所有笔记（通话/短信便签自动排除）→ 通知栏显示进度 → 完成显示"新增 N 条"
- **下载**：点击同步 → 选择"下载（从远程读取）"→ 列出远程目录 → 逐个下载 `.json` 文件 → 增量写入本地（已有笔记更新，新笔记新建，不删除本地独有笔记）→ 完成显示"新增 N 条，更新 M 条"
- **网络容错**：OkHttp 内置 3 次指数退避重试（1s/2s/4s），网络波动时自动恢复
- **前台服务**：同步期间状态栏显示持久通知，防止系统杀死进程
- **数据安全**：所有数据存储在用户自己的 GitHub 私有仓库中，只有 Token 持有者可访问

### 4. 下载并导入依赖库

项目依赖 `httpcomponents-client` 组件（用于 Google Tasks 同步的 HTTP 通信）：

- **下载地址**：https://hc.apache.org/downloads.cgi（下载 4.5.14 版本）
- 解压后，通过 **File → Project Structure → Dependencies → app → + → Jar Dependency**，依次导入 `lib` 目录下的以下 3 个 jar 文件：
  - `httpclient-osgi-4.5.14.jar`
  - `httpclient-win-4.5.14.jar`
  - `httpcore-4.4.16.jar`
- **不要**导入 lib 目录下的所有 jar，否则会出现 jar 包冲突

### 5. 解决常见编译错误

| 错误现象 | 原因 | 解决方法 |
|---|---|---|
| **控制台输出乱码** | IDE 默认编码与项目编码不一致 | 双击 Shift 搜索 `vmoption` → Edit Custom VM Options → 在文件末尾添加 `-Dfile.encoding=UTF-8` → 重启 Android Studio |
| **找不到符号 `setLatestEventInfo`** | 该方法在 Android 6.0（API 23）已废弃并在高版本 SDK 中移除 | 在 `GTaskASyncTask.java` 第 80 行附近，注释掉或删除调用 `setLatestEventInfo` 的代码 |
| **switch 语句报 `Constant expression required`** | 新版 Gradle 中 R 资源 ID 不再是 final 常量 | 在 `gradle.properties` 文件中添加：`android.nonFinalResIds=false` |
| **3 files found with path 'META-INF/DEPENDENCIES'** | 多个 jar 包中包含同名元数据文件 | 已在第 3 步的 `build.gradle` 中通过 `packagingOptions { exclude 'META-INF/DEPENDENCIES' }` 解决 |
| **Gradle 下载依赖缓慢或失败** | 默认 Maven 仓库为 Google 官方源，国内访问不稳定 | 已在第 3 步中配置阿里云镜像源 |

### 6. 同步或更新 Gradle 版本

如果 Gradle 版本不兼容，需手动调整：

- 推荐的 Gradle 版本范围：**7.6 ~ 8.7**
- 推荐的 Android Gradle Plugin（AGP）版本：**7.4.x ~ 8.2.x**
- JDK 版本要求：**JDK 11 及以上**

具体操作：
1. 修改 `gradle/wrapper/gradle-wrapper.properties` 中的 `distributionUrl` 为目标版本
2. 修改项目根目录 `build.gradle` 中的 AGP 版本
3. 点击 **Sync Project with Gradle Files** 重新同步

### 7. 编译构建

1. 点击工具栏的 **小锤子图标**（Make Module 'app'）进行编译
2. 等待 Gradle 构建完成，确认 **Build Output** 窗口显示 `BUILD SUCCESSFUL`
3. 若有编译错误，根据第 5 条中的方法逐一排查解决

### 8. 运行项目

1. 创建虚拟设备：**AVD Manager** → **Create Virtual Device** → 选择设备型号（如 Pixel 6） → 选择系统镜像（API 30+，推荐 API 33） → Finish
2. 或连接 Android 真机（需开启 USB 调试模式）
3. 点击绿色**运行按钮**（Run 'app'），选择目标设备
4. 等待 APK 安装完成，应用将在设备上自动启动

### 部署注意事项

- 由于小米便签开源代码编写年代较早（2010-2011），在新版 Android Studio（2023+）和 AGP 8.x+ 环境下会遇到较多兼容性问题，建议使用 AGP 7.4~8.2 范围
- GitHub 同步功能需要在 GitHub 创建私有仓库 `xiaomibianqian`，并生成 Personal Access Token（勾选 `repo` scope），在应用设置中粘贴 Token 完成配置
- 同步需要前台服务权限（`FOREGROUND_SERVICE` + `FOREGROUND_SERVICE_DATA_SYNC`），已在 `AndroidManifest.xml` 中声明
- 应用首次启动需要授予通话记录、短信、联系人、通知、精确定时闹钟、全屏通知等运行时权限
- 外部 jar 包引用使用绝对路径（Apache HttpClient 组件仅保留编译兼容），换机部署时需调整 `build.gradle.kts` 中的路径；OkHttp 通过 Gradle 依赖管理自动下载
